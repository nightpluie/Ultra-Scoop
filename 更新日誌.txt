v3.4.1  （2026-06-12）
────────────────────────────────────

【修正：快捷鍵時好時壞（中文輸入法／Caps Lock）】
- 過去在中文輸入法（注音等）啟用中或 Caps Lock 開啟時，
  Cmd+C／V／A／G 等快捷鍵會失效，必須改用上方選單操作
- 原因：macOS 上 Tk 以「按鍵產生的字元」比對快捷鍵，
  輸入法或大寫狀態會讓字元變形導致比對不到
- 新版加入萬用攔截層：字元變形時改以「實體按鍵碼」判斷，
  並走與選單列相同的內部路徑，任何輸入法狀態下快捷鍵都有效

【修正：Cmd+V 可能貼上兩次】
- 修正既有的事件攔截寫法錯誤（未正確阻斷系統預設處理），
  過去快捷鍵正常運作時，貼上／剪下可能被執行兩次

v3.4  （2026-06-11）
────────────────────────────────────

【介面簡化：設定集中管理】
- 新增右上角「⚙ 設定」按鈕（快捷鍵 Cmd+,），集中管理：
    API Key、記憶 API Key、Claude 模型、技能包、
    Whisper 模型與語言
- 移除中間欄的「進階設定」摺疊區，主畫面只保留寫稿必要欄位
- 首次啟動若未設定 API Key，自動開啟設定視窗引導

【介面簡化：輸入來源分頁】
- 輸入素材欄改為四個分頁：現場錄音｜手機麥克風｜音檔轉錄｜附件
- 一次只顯示一種輸入方式的控制項，畫面大幅清爽
- 切到「手機麥克風」分頁即自動啟動伺服器並顯示 QR Code，
  切走即自動關閉（原本的開關移除）
- 現場錄音、手機麥克風、音檔轉錄共用同一組 Whisper 模型／語言設定
  （原本音檔轉錄有獨立的模型選單，已合併）

【新功能：查核結果點擊跳轉】
- 事實查核結果中的問題文字可直接點擊，
  右欄稿件自動捲動至對應位置並短暫亮起標記
- 不必再自己在稿件中尋找黃底灰底的位置

【新功能：錄音備份自動清理】
- 啟動時自動檢查 rec/ 資料夾，發現超過 30 天的錄音備份時
  詢問是否刪除（逐字稿文字備份在 log/，不受影響）
- 避免錄音檔長期累積佔滿磁碟

【按鈕改名（更直覺）】
- 「校正」→「修正錯字」
- 「翻譯」→「翻成中文」
- 逐字稿工具列（貼上／清除／載入／儲存／修正錯字／翻成中文）
  移至逐字稿框正上方，位置更合理

【程式架構整理】
- 主程式拆分為多個模組：app_theme（主題）、text_utils（文字解析）、
  claude_api（AI 呼叫）、ui_helpers（介面元件），降低維護難度
- 新增 requirements-lock.txt 記錄已驗證可用的套件版本組合
- 移除程式碼中不再使用的存取金鑰，提升安全性

v3.3  （2026-06-02）
────────────────────────────────────

【公開發行：GitHub 一鍵更新】
- Ultra Scoop 移至公開 repo，雙擊「更新 Ultra Scoop」即可取得最新版，不必每次重傳整包檔案
- 更新時自動保護個人設定，你填入的 API key 不會被覆蓋
- 即使是用壓縮檔複製、沒有 git 紀錄的資料夾，「更新」也會自動掛上更新來源後再更新

【改善：全新 Mac 一鍵安裝】
- ffmpeg 改為隨套件自動安裝（imageio-ffmpeg），不再需要事先安裝 Homebrew 或系統 ffmpeg
- 解決全新電腦上「匯入音訊／影片檔轉錄」與「手機麥克風轉錄」因缺少 ffmpeg 而失敗的問題
- 安裝程式會自動解除 macOS 對外來檔案的開啟限制（Gatekeeper），
  之後雙擊「啟動」「更新」不再被擋（第一次仍需對「安裝」檔按右鍵 → 打開）
- 安裝時自動建立個人設定檔、預先下載 ffmpeg，整包離線後仍可運作

【調整：個人設定與版控分離】
- config.json（含 API key）改為各台電腦各自保留，更新時永不覆蓋
- 新增 config.example.json 範本，安裝時自動複製為 config.json

v3.2  （2026-04-14）
────────────────────────────────────

【修正：校正功能截斷逐字稿】
- 過去長篇逐字稿（約 30 分鐘以上）校正後內容會被截短，原因是 Claude API
  單次輸出有 token 上限，舊版沒有分段處理
- 新版自動將逐字稿切為每段約 3000 字的區塊，逐段校正後合併，
  無論逐字稿多長都不會截斷
- 若極端情況下單段仍超出限制，該段自動保留原文，不會丟失內容

【改進：校正時自動帶入記者會背景資訊】
- 校正階段現在會把「受訪者 / 發言人」欄位以及本場記者會名稱
  一併傳給 AI，作為人名、機構名、專有名詞修正的參考依據
- 有填入背景資訊時，人名與專有名詞的校正準確率明顯提升
- 未填入時，行為與舊版完全相同

【修正：手機麥克風模式 Whisper 幻覺問題】
- 過去將記者會名稱傳給 Whisper 作為 initial_prompt，
  導致 Whisper 把名稱當作已說過的文字、強行插入相關詞彙
- 移除此機制，記者會名稱改僅用於錄音存檔命名
- 錯字修正責任移交至校正階段（由 Claude 處理），效果更精準且可控

【調整：即時轉錄音訊切段時長 5 秒 → 10 秒】
- 每段送 Whisper 辨識的音訊長度從 5 秒延長至 10 秒
- 句子完整度提升，人名與專有名詞辨識準確率改善
- 代價：逐字稿延遲從 5 秒增為 10 秒（畫面上字出現的速度）

v3.1  （2026-04-10）
────────────────────────────────────

【修正：手機麥克風中斷後自動恢復】
- 錄音途中若被 Siri、來電等系統事件佔用麥克風，恢復後會自動重新取得麥克風並繼續錄音
- 中斷前已產生的逐字稿完整保留，恢復後的內容接續追加，存檔時為同一份完整逐字稿

【修正：手機麥克風重新開始錄音時逐字稿重疊】
- 停止後重新開始錄第二段時，介面逐字稿區自動清空（上一段已由系統自動存檔至 log/）
- 避免手動按「儲存」時第一段與第二段內容合併存成同一個新檔

【新功能：錄音自動存檔】
- 每次錄音結束後，音訊自動儲存至程式同目錄的 rec/ 資料夾
- 筆電麥克風：存為 live_時間戳.wav
- 手機麥克風：存為 phone_時間戳_記者會主題.wav（有填主題時自動帶入檔名）
- 格式為單聲道 16-bit PCM WAV，16000 Hz，可直接用 QuickTime 播放
- 存檔失敗（磁碟空間不足等）不影響轉錄流程

v3.0  （2026-04-08）
────────────────────────────────────

【品牌重命名】
- 程式名稱由「記者會殺手」更名為「ULTRA SCOOP」
- 版本號更新至 v3.0

【新功能：配稿模式】
- 生成報導設定欄新增「同時產出配稿」勾選框與「配稿角度」欄位
- 勾選後，一次生成主稿與配稿兩篇
- 第三欄（最終稿件）新增「主稿 / 配稿」Tab 切換，隨時可在兩篇之間來回
- 配稿自動在第一段引入主稿核心事實作為背景，後續聚焦不同面向

【新功能：逐字稿校正】
- 逐字稿欄位新增「校正」按鈕
- 使用 Claude Haiku 自動修正語音辨識錯誤（人名、機構名、專有名詞、數字）
- 只修正明顯辨識錯字，保留原有時間戳記、說話者標記與段落格式

【事實查核升級】
- 查核類型從單一分類擴充為三類：
    typo      — 錯別字、語音辨識錯字
    mismatch  — 數字、日期、姓名職稱與原始素材不符
    unsourced — 事實或數據在原始素材中找不到出處
- 查核結果新增「建議修正值」欄位，有明確答案時自動提供
- 操作流程改為兩步驟：
    「標記問題」— 在最終稿件以顏色標記（黃底可修正、灰底待人工確認）
    「一鍵修正（N 項）」— 自動替換所有有建議值的 typo / mismatch
- max_tokens 從 1024 提升至 2048，支援更長稿件的查核

【其他調整】
- 寫稿 prompt 從財經專用改為通用新聞，不再限定「財經記者」角色
- Topbar 高度微調（48px → 56px），視覺更穩定
